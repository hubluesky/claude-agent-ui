六、发现的潜在问题

1. auto mode canUseTool 逻辑：getAutoDecision('auto') 返回 null 是正确的（让 SDK 分类器处理）。但 resolvePendingForMode() 中 case 'auto' 仍和 bypassPermissions 一起 auto-allow 所有 pending。这意味着切换到 auto mode 时会立即放行所有等待中的审批，而不是让分类器重新评估。这可能不是期望行为。
2. Rate Limit 渲染：当前 rate_limit_event 渲染为普通错误提示。但 SDK 返回的 SDKRateLimitInfo 包含 resetsAt（重置时间）、utilization（用量百分比）、status（allowed_warning vs rejected）等丰富信息。当前可能未区分 warning 和 rejected 状态。
3. Tool Progress 渲染：当前渲染为 spinner + 截断内容。SDK 的 SDKToolProgressMessage 包含 elapsed_time_seconds 字段，可以显示实时耗时。需确认是否已利用此字段。
4. Prompt Suggestion：消息类型 prompt_suggestion 已定义在 messages.ts 中，但 query options 未传 promptSuggestions: true，SDK 不会生成此消息。需要同时启用 option + 添加渲染。
