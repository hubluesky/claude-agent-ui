阶段概览

Phase 1 快速接入

SDK 已提供 API/消息，只需转发+渲染。改动最小，价值最大。

  • ✓ 1.1 Auto Mode 真实接入 (DONE)
  • ■ 1.2 Tool Progress 流式耗时指示
  • ■ 1.3 Rate Limit 事件提示
  • ■ 1.4 Compaction 状态提示
  • ■ 1.5 Session Complete 结果卡片（cost/tokens）
  • ■ 1.6 Account Info 状态栏

Phase 2 消息增强

改善消息渲染质量，让工具结果更可读、更易操作。

  • ■ 2.1 Task 进度卡片（Started/Progress/Notification）
  • ■ 2.2 工具结果折叠/展开
  • ■ 2.3 Diff 渲染组件（Edit/Write 工具结果）
  • ■ 2.4 代码块增强（复制按钮 + 语言标签）
  • ■ 2.5 Prompt Suggestions（对话结束推荐）
  • ■ 2.6 API Retry 提示

Phase 3 会话管理

补齐会话生命周期功能，对接 SDK 会话 API。

  • ■ 3.1 Session Resume（恢复历史会话交互）
  • ■ 3.2 模型列表 + 动态切换
  • ■ 3.3 Effort 自适应（模型支持级别感知）
  • ■ 3.4 Session Fork（从任意消息分叉）
  • ■ 3.5 Permission Rules 持久化

Phase 4 高级功能

Sub-agent 可视化、Context 面板、MCP 管理等进阶能力。

  • ■ 4.1 Sub-agent 可视化（树形 + 进度摘要）
  • ■ 4.2 Context Usage 可视化面板
  • ■ 4.3 MCP 服务器管理面板
  • ■ 4.4 Budget 控制（maxTurns/maxBudgetUsd）
  • ■ 4.5 File Checkpoint / Rewind

Phase 5 体验打磨

纯 UI 层功能，不依赖 SDK，提升日常使用体验。

  • ■ 5.1 消息搜索
  • ■ 5.2 主题系统（深色/浅色）
  • ■ 5.3 快捷键系统
  • ■ 5.4 Session 导出（Markdown/JSON）

依赖关系

Phase 1 → 无前置依赖

纯 SDK 消息转发 + UI 渲染，可立即开始

Phase 2 → 依赖 Phase 1.5（result 卡片）

Task 卡片和 Diff 组件需要消息渲染基础

Phase 3 → 依赖 Phase 1.6（account info）

模型切换需要先有 supportedModels() 数据

Phase 4 → 依赖 Phase 2.1（Task 卡片）

Sub-agent 可视化构建在 Task 系统之上

Phase 5 → 无前置依赖

纯 UI 功能，可与任意阶段并行
