2.1 Task 进度卡片
复杂度: 中 改动文件: 4 SDK 消息: SDKTaskStarted / SDKTaskProgress / SDKTaskNotification

问题

Agent 用 TaskCreate/TaskUpdate 管理工作进度，但 UI 上完全看不到任务列表和进度。

SDK 消息格式

// Task 开始
{ type:'system', subtype:'task_started',
  task_id:'task_1', description:'修复登录bug',
  task_type:'agent', workflow_name:'debug' }

// Task 进度（周期性）
{ type:'system', subtype:'task_progress',
  task_id:'task_1', description:'修复登录bug',
  usage: { total_tokens:5200, tool_uses:3, duration_ms:12000 },
  last_tool_name:'Grep', summary:'正在搜索相关代码...' }

// Task 完成
{ type:'system', subtype:'task_notification',
  task_id:'task_1', status:'completed'|'failed'|'stopped',
  summary:'已修复登录验证逻辑', output_file:'/tmp/...',
  usage: { total_tokens:15000, tool_uses:8, duration_ms:45000 } }

UI 表现

Task Started
▣ 修复登录bug agent · debug
⏱ 刚开始
Task Progress (实时更新同一个卡片)
▣ 修复登录bug
⏱ 12s · 3 tools · 5.2K tokens
正在搜索相关代码... Grep
Task Completed
✓ 修复登录bug completed
45s · 8 tools · 15K tokens
已修复登录验证逻辑
Task Failed
✗ 修复登录bug failed
23s · 5 tools · 8K tokens

实现要点

  • TaskCard 是有状态组件：started → progress（多次更新）→ notification
  • 用 task_id 关联三种消息，更新同一张卡片而不是新增消息
  • 在 messageStore 中维护 taskMap: Map<string, TaskState>
  • 消息流中仅在 task_started 时插入一条消息，后续更新该消息
  • completed 用绿色左边框，failed 用红色，running 用黄色脉冲

web/hooks/useWebSocket.ts路由 task_started/progress/notification 到 messageStore

2.2 工具结果折叠/展开
复杂度: 低 改动文件: 1 纯 UI

问题

长工具结果（如 Grep 返回几十行、Bash 输出大量日志）占满屏幕，挤掉有意义的内容。

UI 表现

折叠状态 (默认，结果 > 10 行)
Grep pattern="TODO" · 23 matches ▼
src/main.ts:12: // TODO: fix this
src/main.ts:45: // TODO: refactor
src/utils.ts:8: // TODO: add tests
展开状态
Grep pattern="TODO" · 23 matches
src/main.ts:12: // TODO: fix this
src/main.ts:45: // TODO: refactor
src/utils.ts:8: // TODO: add tests
src/api.ts:22: // TODO: error handling
src/api.ts:67: // TODO: retry logic
src/db.ts:11: // TODO: migration
... (23 行)

实现规则

  • 工具结果 ≤ 10 行：完整展示，无折叠按钮
  • 工具结果 > 10 行：默认折叠，显示前 3 行 + "展开全部 (N 行)" 按钮
  • 点击展开/折叠切换，使用 <details> 或 local state
  • 不影响消息流的虚拟滚动（Virtuoso）— 使用 CSS transition 避免跳变

2.3 Diff 渲染组件
复杂度: 中 改动文件: 3 纯 UI

问题

Edit/Write 工具的结果是纯文本，看不出改了什么。claude-code 有语法高亮的 unified diff。

UI 表现

Diff View — Edit Tool
Edit src/agent/v1-session.ts ▼
257   private getAutoDecision(
258 -   case 'auto':
259 -   case 'bypassPermissions':
258 +   case 'bypassPermissions':
260      return { behavior: 'allow' }
+1 -2

方案

从 Edit 工具的 old_string / new_string 输入参数生成 unified diff，然后渲染。

实现细节

  • 使用 diff npm 包（轻量）计算行级差异
  • 删除行：bg-[#f8717110] text-[#f87171]，前缀 -
  • 新增行：bg-[#3fb95010] text-[#3fb950]，前缀 +
  • 上下文行：正常颜色，前缀空格
  • 显示行号（参考 Edit 工具的 file_path）
  • Write 工具：整个 content 都是新增（全绿），可显示前 20 行

文件变更

文件改动
web/package.json新增 diff 依赖
web/components/chat/DiffView.tsx新增：Diff 渲染组件
web/components/chat/MessageComponent.tsxEdit/Write tool_use 块使用 DiffView 渲染
2.4 代码块增强
复杂度: 低 改动文件: 1 纯 UI

问题

Markdown 中的代码块只能看，无法快速复制。

UI 表现

代码块 + Copy 按钮
typescript
function isPrime(n: number) {
  if (n <= 1) return false
  for (let i = 2; i * i <= n; i++)
    if (n % i === 0) return false
  return true
}
点击 Copy 后 → ✓ Copied! 1.5s 后恢复

实现

  • 在 MarkdownRenderer.tsx 的 code 块渲染器中增加 header bar
  • 左侧显示语言标签（从 fence info string 提取）
  • 右侧 Copy 按钮，使用 navigator.clipboard.writeText()
  • 复制成功显示 "Copied!" 1.5s 后恢复
  • 按钮仅在 hover 代码块时显示（避免视觉噪音）

文件改动
web/components/chat/MarkdownRenderer.tsx代码块 header + Copy 按钮
