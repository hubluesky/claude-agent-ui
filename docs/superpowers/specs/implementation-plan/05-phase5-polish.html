5.1 消息搜索
复杂度: 中 改动文件: 4 纯 UI + 本地过滤

问题

长对话中找不到之前讨论过的内容，只能手动滚动。

UI 表现

authentication
3/12 matches
Sure, I'll look into the login flow...
I've updated the authentication module, including the authentication token validation logic and session refresh...
match matched text focus current focus

实现要点

  • 搜索范围：当前 session 已加载的消息（messageStore.messages）
  • 搜索逻辑：简单字符串匹配（case-insensitive），不需要正则
  • 匹配导航：上/下箭头在匹配项间跳转，自动滚动到对应消息
  • 高亮：在 MarkdownRenderer 和 MessageComponent 中用 <mark> 包裹匹配文本
  • 性能：debounce 300ms 输入，避免逐字搜索
  • Esc 关闭搜索栏

5.2 主题系统
复杂度: 中 改动文件: 8+ 纯 CSS + Store

问题

仅有暗色主题，白天使用刺眼。

方案

用 CSS 变量定义主题 token，TailwindCSS 4 原生支持 @theme。通过切换 <html data-theme="light"> 实现。

UI 表现 — 主题切换

Theme

颜色 Token 体系

// CSS 变量

  --color-bg-primary: #0d1117;
  --color-bg-surface: #161b22;
  --color-bg-elevated: #1c2333;
  --color-border: #30363d;
  --color-text-primary: #e6edf3;
  --color-text-secondary: #8b949e;

:root[data-theme="light"] {
  --color-bg-primary: #ffffff;
  --color-bg-surface: #f6f8fa;
  --color-bg-elevated: #ffffff;
  --color-border: #d0d7de;
  --color-text-primary: #1f2328;
  --color-text-secondary: #656d76;
  --color-text-tertiary: #8b949e;
  --color-accent: #b45309;
  --color-success: #1a7f37;
  --color-warning: #9a6700;
  --color-error: #d1242f;
}

实现策略

  • 将所有组件中的硬编码颜色（如 bg-[#1a1918]）替换为 CSS 变量引用
  • 这是最大的改动量 — 需要遍历所有组件更新颜色
  • 分两步：先定义变量并替换核心组件，再逐步替换其余组件
  • System 模式监听 prefers-color-scheme media query
  • 主题偏好存 localStorage + settings DB
