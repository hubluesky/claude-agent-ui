4.1 Sub-agent 可视化
复杂度: 高 改动文件: 7 SDK: listSubagents(), getSubagentMessages(), SDKTaskStarted/Progress/Notification

问题

Agent 频繁 spawn 子 agent，用户完全看不到有多少个子 agent 在运行、各自在做什么、进度如何。

UI 表现 — 内联 Agent 卡片

消息流中 Agent 工具调用
▸ Agent explore-codebase
"搜索认证模块的实现"
⏱ 8s 4 tools Grep
正在分析 auth middleware...
▾ 查看详细消息

UI 表现 — 展开后的子 Agent 消息

Agent 卡片展开状态
▸ Agent explore-codebase completed
15s · 8 tools · 12K tokens
Sub-agent 消息
🔍
Grep "authenticate" → 5 files
📄
Read auth/middleware.ts
📄
Read auth/session.ts
💬
认证使用 JWT + Redis 缓存
▴ 收起

实现要点

  • 启用 agentProgressSummaries: true 获取周期性进度摘要
  • 通过 SDKTaskStarted 的 task_type: 'agent' 识别 agent 类型任务
  • Agent 卡片初始显示进度摘要（来自 SDKTaskProgress.summary）
  • 展开时懒加载子 agent 消息：调用 getSubagentMessages(sessionId, agentId)
  • 子 agent 消息用缩进 + 左边框区分层级
  • 嵌套 agent（agent 启动的 agent）用递增缩进

4.5 File Checkpoint / Rewind
复杂度: 中 改动文件: 5 SDK: enableFileCheckpointing + rewindFiles(messageId)

问题

Agent 改了不满意的代码，想撤回到之前某个节点的文件状态。

SDK API

// 启用
query({ options: { enableFileCheckpointing: true } })

// 撤回
const result = await query.rewindFiles(userMessageUUID, { dryRun: false })
// → { canRewind: true, filesChanged: 3, insertions: 45, deletions: 12 }

UI 表现

用户消息 hover 状态
You
帮我重构这个函数
点击后确认弹窗
↶ Rewind Files
将恢复到此消息发送时的文件状态：
3 files changed
+45 -12 lines
This action cannot be undone. Files will be restored to their previous state.

流程

  1. hover 用户消息 → 显示 Rewind 按钮
  2. 点击 → 先 dryRun: true 获取影响范围
  3. 显示确认弹窗（文件数、行数变化）
  4. 确认 → dryRun: false 执行撤回
  5. 成功 → toast 提示 "Files rewound to checkpoint"
