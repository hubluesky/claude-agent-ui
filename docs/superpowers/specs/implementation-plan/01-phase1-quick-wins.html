1.2 Tool Progress 流式耗时指示
复杂度: 低 改动文件: 4 SDK 消息: SDKToolProgressMessage

问题

Bash 命令执行时用户看到的是静态 spinner，不知道已经运行了多久，也不知道是否卡住。

SDK 消息格式

{
  type: 'tool_progress',
  tool_use_id: 'toolu_xxx',
  tool_name: 'Bash',
  elapsed_time_seconds: 12.5,
  parent_tool_use_id: null,
  task_id?: 'task_xxx'
}

方案

Server 端

文件改动
shared/protocol.ts新增 S2C_AgentMessage 中支持 tool_progress 类型（已有，确认转发）
server/ws/handler.ts确认 session.on('message') 转发 tool_progress 类型消息

Web 端

文件改动
web/hooks/useWebSocket.ts在 agent-message handler 中识别 tool_progress 类型，更新 messageStore
web/components/chat/MessageComponent.tsx渲染 tool_progress 消息为耗时指示器

UI 表现

Tool Progress - Bash 执行中
Bash npm install 12s
<10s 10-30s >30s

设计要点：

  • 在现有工具执行 spinner 旁显示实时耗时（每次收到 tool_progress 更新）
  • 颜色分级：<10s 绿色, 10-30s 黄色, >30s 橙色
  • 显示工具名 + 输入摘要（Bash 显示命令，Edit 显示文件路径）
  • 工具完成后（收到 tool result），耗时指示器消失，替换为结果

实现细节

MessageComponent.tsx 中已有 case 'tool_progress' 渲染路径。需要增强：

  • 用 tool_use_id 关联到对应的工具调用块
  • 实时更新 elapsed 数字（不新增消息，更新现有工具块的状态）
  • 使用 messageStore 中的 streaming 机制或维护独立的 progress map
1.3 Rate Limit 事件提示
复杂度: 低 改动文件: 3 SDK 消息: SDKRateLimitEvent

问题

API 限流时用户不知道为什么响应停了，以为是 bug 或网络问题。

SDK 消息格式

{
  type: 'rate_limit_event',
  rate_limit_info: {
    status: 'allowed' | 'allowed_warning' | 'rejected',
    resetsAt?: 1712150400000,    // Unix ms
    rateLimitType?: 'five_hour' | 'seven_day' | ...,
    utilization?: 0.85,          // 0-1 百分比
    isUsingOverage?: false,
    surpassedThreshold?: 80
  }
}

方案

Server 端

转发 rate_limit_event 消息到客户端（与其他 SDK 消息相同路径）。

UI 表现 — 三种状态

allowed_warning - 用量警告 Banner
API 用量 85%
5 小时配额将在 14:30 重置
rejected - 限流阻断 Banner
API 限流 — 等待重置中
12:34
输入消息...
Composer 被禁用，直到限流重置
allowed (utilization) - 状态栏用量指示
claude-sonnet-4-6 | API
80%

文件变更

文件改动
shared/messages.tsAgentMessage 联合类型增加 rate_limit_event
web/hooks/useWebSocket.ts识别 rate_limit_event，存入 connectionStore
web/components/chat/RateLimitBanner.tsx新增：限流 banner 组件，含倒计时
web/components/chat/ChatInterface.tsx在消息区顶部渲染 RateLimitBanner

实现细节

  • connectionStore 新增 rateLimitInfo: SDKRateLimitInfo | null
  • 倒计时用 setInterval + resetsAt - Date.now() 计算剩余时间
  • rejected 状态下禁用 ChatComposer 的发送按钮
  • 重置后自动清除 banner
