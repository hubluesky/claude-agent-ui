分类器决策流程

工具请求 (canUseTool) │ ├─ 安全白名单? ── YES ──▶ 直接放行 (0ms, 无 API 调用) │ FileRead, Grep, Glob, LSP, TaskList, TaskGet, TodoRead, │ AskUserQuestion, EnterPlanMode, SendMessage, Skill ... │ NO │ ├─ Stage 1: 快速分类 │ Claude API · 64 tokens · temperature=0 · stop=['</block>'] │ ├─ <block>no</block> ──▶ 放行 │ ├─ <block>yes</block> ──▶ 升级 Stage 2 │ └─ 解析失败 ──▶ 升级 Stage 2 │ ├─ Stage 2: 深度推理 │ Claude API · 4096 tokens · temperature=0 · <thinking> 推理 │ ├─ <block>no</block> ──▶ 放行 │ ├─ <block>yes</block> ──▶ 拦截 → 弹审批给 lock holder（可覆盖） │ └─ 解析失败 ──▶ 拦截 (fail-closed) │ └─ 错误降级: API 失败/超时 → 降级为用户手动审批

设计决策（已确认）： API Key 复用 SDK 的 key · 模型与主模型相同 · 两阶段完整复刻 · 拦截但可覆盖（lock holder 可强制放行）
