两个全新方案——基于第二轮深度调研

之前的设计缺少 Workspace 面板（文件变更、终端输出、任务进度），也没有利用看板自动流转的概念。这次综合了 Devin 三栏 Workspace、Nimbalyst 看板流转、Warp 通知机制。

灵感来源：Devin 2.0 Workspace · Nimbalyst Kanban · Cursor 3 Agent Tabs · Warp 通知系统 · GitHub Mission Control
A

Pros

  • 看板泳道一眼看到哪些等待、运行、完成
  • 会话自动流转——完成了自动去 Done 列
  • Workspace 面板看到文件变更+终端+进度

Cons

  • 泳道卡片多时水平可能拥挤

垂直看板列：Nimbalyst 风格全屏看板

整个页面就是看板。每列代表一个状态（Waiting / Running / Review / Done），会话卡片在列间自动流转。卡片内嵌消息预览、进度、直接审批按钮。点击卡片展开为浮动聊天窗口。
