2. 权限模式系统

当前：ModesPopup + 服务端
● Ask before edits default
● Auto-accept edits acceptEdits
● Bypass permissions bypass
● Plan mode plan

❌ 缺失

  • bypassPermissions 无安全检查
  • 无 prePlanMode 保存/恢复
  • 进入 plan 模式时不记住之前模式
  • 退出 plan 时不恢复之前模式
  • ModesPopup 按钮文字为英文
目标：增强的模式系统
● 编辑前询问 default
● 自动接受编辑 acceptEdits
● 跳过所有权限 ⚠ 仍检查安全规则
● 计划模式 退出时恢复之前模式

✅ 改进

  • ModesPopup 文字中文化
  • bypassPermissions: 安全工具仍走审批
  • prePlanMode: 进入 plan 时保存当前模式
  • 退出 plan 时自动恢复 prePlanMode

3. 代码改动汇总

AskUserPanel 改动

AskUserPanel.tsx multiSelect 模式：checkbox 样式 + 多选 toggle + "确认选择" 按钮
preview 支持：选项带 preview 时分栏展示
shared/tools.ts AskUserRequest 确认 multiSelect / preview 字段已定义
v1-session.ts handleAskUserTool: 透传 multiSelect, preview 到事件

权限模式改动

ModesPopup.tsx 按钮文字中文化；bypassPermissions 加安全提示
v1-session.ts getAutoDecision('bypassPermissions'): 安全敏感工具（修改 .git/.claude/shell config）仍走审批
新增 _prePlanMode 字段：进入 plan 时保存、退出时恢复
handler.ts handleSetMode: 进入 plan 时保存 prePlanMode；退出时恢复
shared/constants.ts SAFETY_SENSITIVE_PATTERNS: 定义 bypass 仍需审批的工具/路径模式
