1. 组件架构

统一审批框架层级
ApprovalPanel 新组件 — 统一容器
● claim-lock 逻辑
canClaim / canInteract / handleClaim
● readonly 展示
等待操作者响应 / 锁超时恢复
● 统一样式
圆角容器 / 编号选项 / 键盘快捷键
▼ 通过 config 驱动渲染 ▼
ToolApprovalConfig
  • title: 工具名 + 描述
  • content: 工具输入展示
  • options: 允许/始终允许/拒绝
  • feedbackField: 拒绝原因
  • onDecision → respondToolApproval()
PlanApprovalConfig
  • title: 接受此计划？
  • content: 计划内容 (Markdown)
  • options: 动态生成（基于模式/上下文%）
  • badge: 上下文占比
  • feedbackField: 修改建议
  • onDecision → respondPlanApproval()
AskUserConfig
  • title: Claude 需要输入
  • content: 问题文本
  • options: SDK 预设选项
  • multiSelect: 多选模式
  • preview: 选项预览
  • otherField: 自定义输入
  • onDecision → respondAskUser()

2. ApprovalPanel 接口设计

TypeScript 接口
interface ApprovalOption {
  key: string           // 决策标识 'allow' | 'deny' | 'auto-accept' ...
  label: string         // 显示文字
  description?: string   // 选项说明
  color?: 'green' | 'amber' | 'purple' | 'gray' | 'red'
  preview?: string       // AskUser: 选项预览内容
}

interface ApprovalPanelConfig {
  type: 'tool-approval' | 'plan-approval' | 'ask-user'
  title: string           // 面板标题
  titleIcon?: ReactNode   // 标题图标
  badge?: string          // 右上角 badge（如"上下文 42%"）
  content?: ReactNode    // 中间区域（工具输入/计划内容/问题文字）
  options: ApprovalOption[]
  multiSelect?: boolean  // AskUser 多选模式
  feedbackField?: {       // 可选文本输入
    placeholder: string
    submitKey?: string    // 关联的决策 key（按 Enter 时触发）
  }
  otherField?: {          // AskUser "其他" 输入
    placeholder: string
  }
  onDecision: (key: string, payload?: any) => void
  readonly: boolean
  canClaim: boolean
}
