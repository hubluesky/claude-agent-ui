方案对比：Tauri 桌面端 vs 浏览器 + 系统托盘

同样的功能目标，两种完全不同的实现路径

方案 A：Tauri 桌面端
┌─────────────────────┐
│  Tauri App (Rust)    │
│  ├── WebView         │
│  │   └── Web UI      │
│  ├── 系统托盘        │
│  └── Server 子进程   │
└─────────────────────┘
技术栈新增
Rust + Tauri 2.0 + Cargo 工具链
新代码量
~300-500 行 Rust（托盘 + sidecar + IPC commands）
+ tauri.conf.json + Cargo.toml 配置
构建产物
.msi / .dmg / .AppImage 安装包
含 WebView2(Win) 或 WebKit(Mac/Linux) 依赖
安装包大小
~10-20MB（不含 Node.js）
UI 渲染
内嵌 WebView 窗口，独立于浏览器
用户感知
一个独立的桌面应用程序
方案 B：浏览器 + 系统托盘
┌─────────────────────┐
│  Node.js Server     │
│  ├── Fastify (API)   │
│  ├── 系统托盘        │
│  └── 静态文件 (UI)   │
└─────────────────────┘
        ↑
  浏览器访问 localhost
技术栈新增
systray2（npm 包，Node.js 内调用）
或一个 ~100 行的 Rust 小工具管理 Node 进程
新代码量
~50-100 行 TypeScript（托盘集成到 Server）
+ 管理面板 React 页面（两种方案都要写）
构建产物
和现在一样：node dist/index.js
无需额外安装步骤
安装包大小
~5MB（纯 JS + 小型托盘二进制）
UI 渲染
用户默认浏览器，和现在一样
用户感知
一个在系统托盘里的后台服务 + 浏览器页面

逐项对比

维度 Tauri 桌面端 浏览器 + 托盘
开发成本 高 — 学 Rust + Tauri API 低 — 纯 TypeScript
维护成本 高 — Rust 编译、Tauri 版本、跨平台 WebView 低 — 现有技术栈
防误杀 好 — 独立应用名，非 node.exe 中 — 托盘有图标可识别，但任务管理器里仍是 node.exe
防终端关闭 完全解决 — 不依赖终端 完全解决 — 托盘模式不依赖终端
系统托盘 原生 — Tauri 内置 依赖第三方 — systray2 或小型 Rust 工具
管理面板 UI Web UI（WebView 内） — 两种方案相同 Web UI（浏览器内） — 两种方案相同
SDK 更新 相同 — node_modules 替换 相同 — node_modules 替换
开发模式体验 Tauri dev 窗口 + HMR 浏览器 + HMR — 和现在一样
跨平台 好 — Tauri 原生支持三平台 需验证 — systray2 跨平台兼容性待确认
后续 Mobile 端 有路径 — Tauri Mobile 可复用 无关 — Mobile 仍需另外做
开机自启 内置 — Tauri autostart 插件 需实现 — 注册表 / launchd / systemd
开发到上线时间 长 — 2-3 周（含学习 Rust） 短 — 3-5 天

核心取舍

选 Tauri 的理由
  • 任务管理器里有独立应用名，不是 node.exe
  • P2 规划已包含 Tauri，迟早要做
  • 为后续 Mobile 端铺路
  • 开源项目更 professional 的分发体验
  • 开机自启、auto-updater 等系统集成开箱即用
选浏览器+托盘的理由
  • 开发成本极低，几天就能上线
  • 零新技术栈，纯 TypeScript
  • 管理面板的 Web UI 两种方案都要写，共享 90% 工作量
  • 可作为 Tauri 的前置方案：先上托盘解决痛点，后续再包 Tauri 壳
  • 更少的维护负担

💡 可以渐进式推进

两种方案不互斥。90% 的工作量（管理面板 Web UI、SDK 更新逻辑、日志/配置/状态监控）是共享的。差异只在「窗口壳」这一层：

Phase 1（现在）：浏览器 + 系统托盘 → 快速解决痛点
Phase 2（后续）：包 Tauri 壳 → 升级为独立桌面应用

Phase 1 开发的管理面板 UI、Server 管理逻辑、SDK 更新机制，在 Phase 2 中 100% 复用。Tauri 只是加了个窗口壳和更好的系统集成。
