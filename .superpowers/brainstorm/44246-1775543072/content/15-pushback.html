5 个设计问题 + 改进建议

左边是当前设计的问题，右边是建议的改进方案。每个都有具体的 mockup 说明为什么当前方案有问题。

1
三种模式多余 → 只需要两种
当前设计有 Single / Focus / Multi 三种模式。但 Focus 和 Single 的界面完全一样（项目树侧边栏 + 完整聊天），唯一区别是 Focus "记得自己从 Multi 来"可以 Esc 返回。这不需要一个独立的模式。
当前设计三种模式

Focus
Multi

问题：用户点 Focus 和 Single 看到的是一模一样的界面。
他会困惑："这两个有什么区别？"

用户需要理解三个模式的含义，但其中两个看起来完全一样。认知负担高，选择困难。

建议改进两种模式 + 返回按钮

只有两个选择，清晰明确。

从 Multi 展开面板后：

← 返回 Multi

TopBar 出现"返回 Multi"按钮。Esc 也能返回。
不是新模式，只是一个临时状态。

两个模式 = 简单。展开面板就是临时切到 Single 聊天，TopBar 多一个"← 返回 Multi"按钮。

2
8 个 Virtuoso 实例 → 性能灾难
当前 ChatMessagesPane 用了 Virtuoso 虚拟滚动组件（重型）。MiniChatPanel 复用它意味着 8 个面板 = 8 个 Virtuoso 实例同时运行，内存和渲染开销巨大。

每个 Virtuoso 实例都有自己的 DOM 测量、ResizeObserver、滚动事件。6-8 个同时运行会明显卡顿。

Mini 面板空间小，不需要虚拟滚动。用 overflow-y: auto 的 div，只渲染最近 20 条消息。展开到全屏（Single）时才用 Virtuoso。

3
WebSocket multi-join 重构太大 → 用多连接更简单
当前 WSHub 是"一个连接 = 一个 session"。改成 multi-join 需要重构 WSHub（sessionId → Set）、handler（lock 逻辑）、connectionStore（全部改为 per-session Map）。这是一个巨大的改动，容易引入 bug。
当前设计Multi-join 重构
Browser
↕ 1 个 WebSocket
Server

需要改动的文件：

❌ hub.ts — sessionId → Set<string>
❌ handler.ts — join 不再 leave
❌ protocol.ts — 新消息类型
❌ connectionStore — 全部改为 per-session Map
❌ useWebSocket.ts — 消息路由到对应面板

5 个文件的深度重构。connectionStore 的 pendingApproval、lockStatus 等全是单值，改成 per-session Map 会影响所有消费这些状态的组件。

建议改进每面板独立 WebSocket
Browser
↕ N 个 WebSocket（每面板 1 个）
Server

需要改动的文件：

✅ hub.ts — 不改
✅ handler.ts — 不改
✅ protocol.ts — 不改
✅ connectionStore — 不改
✅ 新建 usePanelWebSocket hook — 轻量版 useWebSocket

每个 MiniChatPanel 创建独立 WebSocket 连接，join 自己的 session。服务端零改动。8 个 WebSocket 连接对现代浏览器零压力。每个面板的锁/审批/状态天然隔离。

4
自动弹面板太混乱 → 改为通知条
当前设计：会话变 waiting → 自动弹出新面板；会话变 idle → 10 秒后自动关闭面板。当多个会话同时变状态时，面板不停出现/消失，非常混乱。
当前设计自动弹出/关闭面板

场景：3 个会话同时需要审批

碰撞检测 弹出!
登录流程 弹出!
E2E 测试 弹出!
面板不停跳出来 💥

场景：2 个会话完成

面板突然开始消失，你还没看完输出就没了

自动弹出打断你正在做的事。自动关闭让你来不及看完结果。用户失去对界面的控制感。

建议改进通知条 + 手动添加

场景：3 个会话需要审批

3 个会话需要操作 添加到面板

一条安静的通知，用户决定什么时候添加

完成的面板：

重构 UI (running)
WebSocket (running)
认证中间件 ✓

完成的面板降低透明度排到最后，但不消失。用户手动关闭。

通知条不打断工作流。完成的面板保留让你看输出。用户始终掌控面板。

5
Multi 侧边栏看不到未添加的会话
Multi 模式侧边栏只显示面板中的会话。如果你有 8 个会话但只打开了 3 个面板，侧边栏看不到另外 5 个。需要点"添加"按钮才能看到它们。
