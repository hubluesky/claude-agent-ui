结论 弹窗在 Single 模式不需要
Single 模式侧边栏已经显示所有会话的实时状态（Waiting 置顶+进度条+消息预览）。弹窗只在 Multi 模式才需要——因为 Multi 模式主区域被面板网格占满了，需要一个浮窗看汇总。

Single 模式

  • 侧边栏 = 扁平会话列表（跨项目）
  • 状态筛选：All / Running / Waiting / Idle
  • Waiting 自动置顶 + 黄色背景
  • 每条显示：状态点+标题+项目+进度
  • 不需要弹窗——侧边栏够用了

Multi 模式

  • 侧边栏也是扁平列表（和 Single 一样）
  • 主区域 = N 面板网格
  • 右下角弹窗 = 面板会话汇总（可选）
  • 从侧边栏点击添加会话到面板
  • 侧边栏标注哪些已在面板中

共用同一个侧边栏

  • 两种模式侧边栏完全一样
  • 不需要 MultiSidebar 组件
  • 不需要 AppLayout 条件渲染
  • 只需升级现有 SessionList
  • 改动量最小
