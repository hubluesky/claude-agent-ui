开发模式 vs 生产模式 — 技术方案对比

同一个 Tauri App，两种完全不同的运行方式

🛠️ 开发模式 (pnpm dev)
WebView 来源
http://localhost:5173
Vite dev server，支持 HMR 热更新
Server 进程
外部进程 — tsx watch src/index.ts
由 Turborepo 启动，与 Tauri 无关
代码改了自动重启
端口
固定：Server 4000 / Vite 5173
Tauri 角色
仅提供窗口 + 系统托盘
不管理 Server 生命周期
关闭 Tauri 窗口不影响 Server
SDK 更新
pnpm update @anthropic-ai/claude-agent-sdk
直接用包管理器，和普通开发一样
Rust 代码热更新
cargo watch via tauri dev
修改 Rust 代码自动重编译
启动命令
pnpm dev — 并行启动三个进程
turbo: server(tsx) + web(vite) + desktop(tauri dev)
📦 生产模式 (安装包)
WebView 来源
内嵌静态文件 dist/web/
Vite build 产物，打包进 Tauri bundle
Server 进程
Sidecar — Node.js SEA 单文件可执行
Tauri Rust 主进程管理启停
App 启动时自动拉起，退出时自动关闭
端口
动态分配：Server 找空闲端口
Rust 侧检测端口后注入 WebView
Tauri 角色
完整管理者 — 窗口 + 托盘 + sidecar
关闭窗口 → 最小化到托盘（Server 继续）
退出 App → 停止 Server sidecar
SDK 更新
GUI 内触发：下载新 sidecar 二进制 → 替换 → 重启
或：整体 App 更新（auto-updater）
Rust 代码
编译为原生二进制，随安装包分发
构建命令
pnpm build && pnpm tauri build
产出 .msi / .dmg / .AppImage 安装包

关键技术细节

Node.js SEA (Single Executable Application)

Node.js 22+ 原生支持将脚本打包为单文件可执行。不需要用户安装 Node.js。

node --experimental-sea-config sea-config.json

产出 server.exe (~50-80MB)，含 Node.js 运行时 + server 代码 + node_modules

Tauri Sidecar 机制

在 tauri.conf.json 中声明 sidecar 二进制。Tauri 自动处理路径、平台后缀、权限。

"bundle": { "externalBin": ["binaries/server"] }

Rust 侧通过 Command::new_sidecar("server") 启停

端口协商

生产模式下端口动态分配的流程：

  1. Rust 找一个空闲端口
  2. 通过 CLI 参数传给 sidecar: --port 54321
  3. Server 启动后 stdout 输出就绪信号
  4. Rust 读到信号后，WebView 加载 http://localhost:54321
tauri.conf.json 开发配置

Tauri 原生支持 dev/build 分离配置：

"build": { "devUrl": "http://localhost:5173", "frontendDist": "../web/dist" }

dev 时用 devUrl，build 时用 frontendDist
